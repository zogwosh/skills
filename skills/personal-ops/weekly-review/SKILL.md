# Weekly Review

## Purpose

Generate a structured weekly review that clarifies progress, exposes blockers, and defines the next week's concrete priorities.

## Inputs

- Weekly notes, task logs, calendar highlights, and completed items.
- Current goals or quarterly objectives.
- Optional constraints such as available time or pending dependencies.

## Workflow

1. Group the week's work by goal or project outcome.
2. Distinguish completed deliverables from in-progress effort.
3. Identify blockers, root causes, and unresolved decisions.
4. Propose next actions with clear scope and priority.
5. Provide a compact review for reflection and planning.

## Output Format

- `Wins` with measurable outcomes.
- `Blockers` with proposed mitigations.
- `Lessons` describing process improvements.
- `Next Week Plan` with prioritized action items.

## Edge Cases

- Sparse notes: derive a minimum review from calendar and task residue.
- Too many parallel efforts: force rank by impact and urgency.
- Repeated blockers: escalate by turning them into explicit decisions.
