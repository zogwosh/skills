# Blog Editor

## Purpose

Improve blog drafts so they read clearly, stay technically accurate, and match the target audience's context and expectations.

## Inputs

- Draft article text and target audience.
- Publishing goal (education, launch, case study, etc.).
- Constraints such as tone, length, or required references.

## Workflow

1. Audit structure: headline, narrative flow, section sequence, and conclusion.
2. Tighten language for clarity, precision, and consistency.
3. Fix factual or logical gaps and call out unverifiable claims.
4. Improve transitions, examples, and call-to-action relevance.
5. Return an edited draft with major rationale notes.

## Output Format

- `Edited Draft` with improved wording and structure.
- `Major Edits` list describing high-impact changes.
- `Fact Check Notes` identifying uncertain or unsupported statements.
- `Optional Variants` for title and intro hook.

## Edge Cases

- Mixed audiences: provide a baseline version plus one technical-depth variant.
- Overly promotional copy: rebalance with concrete evidence and examples.
- Missing conclusion: add explicit takeaway and next action.
