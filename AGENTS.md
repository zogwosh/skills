# AGENTS.md (skills scope)

This guide applies to everything under `skills/`.

## Project structure

The source of truth in this repository is `skills/`, not `.agents/skills/`.

- `skills/<category>/<skill-name>/SKILL.md`
  - Canonical skill content in this repo.
- `templates/basic-skill/SKILL.md`
  - Starter template for new skills.
- `scripts/validate.ts`
  - Enforces path and frontmatter rules.
- `scripts/generate-registry.ts`
  - Regenerates `registry.json` from `skills/`.
- `scripts/generate-readme.ts`
  - Regenerates the skills list block in `README.md`.
- `scripts/install-codex.ts`
  - Copies each skill from `skills/` to `~/.agents/skills/<skill-name>/`.
- `scripts/install-claude-code.ts`
  - Copies each skill from `skills/` to `~/.claude/skills/<skill-name>/`.

## Add a new skill (best-practice workflow)

1. Create a directory:
   - `skills/<category>/<skill-name>/`
2. Copy template:
   - `templates/basic-skill/SKILL.md` -> `skills/<category>/<skill-name>/SKILL.md`
3. Fill frontmatter correctly:
   - `name` must equal `<skill-name>`
   - `name` must be lowercase letters, numbers, and hyphens only
   - `description` must be explicit and at least 40 chars
4. Write `description` with clear trigger boundaries:
   - Include when the skill should be used
   - Include when it should not be used
5. Keep one skill focused on one job:
   - Keep `SKILL.md` concise and actionable
   - Put large reference content in sibling folders like `references/`, `scripts/`, `assets/`, `agents/`
6. Do not add extra `SKILL.md` files under nested folders.
   - Only one `SKILL.md` per skill directory.
7. Run checks from repo root:
   - `bun run check`
8. If needed, install locally:
   - Codex: `bun run install:codex`
   - Claude Code: `bun run install:claude`

## Required conventions

- Every installable skill must live under `skills/`.
- Skill path format is fixed: `skills/<category>/<skill-name>/SKILL.md`.
- Skill names must be unique across all categories.
- Keep `registry.json` and `README.md` generated by scripts, not manual edits.

## Codex skills best-practice requirements

Based on https://developers.openai.com/codex/skills.

- Keep each skill focused on one job.
- Prefer instruction-only skills; add `scripts/` only when deterministic behavior or external tooling is required.
- Write imperative instructions with explicit inputs and outputs.
- Validate trigger behavior with prompt checks:
  - Positive prompts should activate the skill.
  - Negative prompts should not activate the skill.
- Optimize `description` for implicit invocation:
  - Keep scope boundaries explicit (`should trigger` / `should not trigger`).
  - Front-load primary use case and trigger keywords so matching still works when descriptions are shortened.
- Respect progressive disclosure constraints:
  - Keep `name` and especially `description` concise and information-dense.
  - Avoid bloated descriptions because initial skill listing context is capped.
- Keep large reference content outside `SKILL.md` in sibling folders (`references/`, `assets/`, `scripts/`, `agents/`).
- Use plugins for reusable distribution:
  - Local/repo authoring stays as direct skill folders.
  - Cross-repo or packaged distribution should use plugins.
- If a skill should be explicit-only, set `agents/openai.yaml` policy `allow_implicit_invocation: false`.
